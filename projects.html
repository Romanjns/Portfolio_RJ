Shopmore E-commerce Security Check (Individual Project)

Application Security Penetration Testing

Context & Background

For this solo school project in Application Security, I performed a full penetration test on "Shopmore," a vulnerable web application provided by my instructor. The goal was to find security weaknesses, demonstrate their impact, and write a detailed report for developers and decision-makers to improve the platform's security.

Concrete Results

  • Critical Findings: Missing anti-forgery tokens on the login page, unauthenticated admin access.
  • High-Risk Findings: Persistent Cross-Site Scripting (XSS) in product creation and orders page.
  • Medium Findings: SQL exceptions from missing inputs, clickjacking vulnerabilities.
  • Security Report: Delivered a detailed technical report including risk severity, CVSS scoring, reproduction steps, and mitigation strategies.

Tools Used

Burp Suite OWASP ZAP CVSS Calculator OWASP WSTG

My Role

As the sole tester, I handled every stage of the penetration test:

  • Scoping and setting test accounts and environment (Virtual Machine setup).
  • Performing manual and automated security tests following OWASP Top 10 and WSTG guidelines.
  • Exploiting vulnerabilities and documenting clear proof of concept for each finding.
  • Prioritizing risks based on CVSS scores and writing mitigation recommendations.
  • Creating a professional 7-page security report to present the results.

What I Learned

Penetration Testing Process

Learned how to conduct security testing from scoping to final reporting.

Finding and Exploiting Vulnerabilities

Gained experience with real-world attack techniques such as XSS, CSRF exploitation, and privilege escalation.

Security Reporting

Improved my ability to document vulnerabilities clearly and professionally for technical and non-technical audiences.

Using Professional Security Tools

Strengthened skills with Burp Suite, OWASP ZAP, and CVSS-based risk evaluation.

IoT Thermostat with Enhanced Cooling System and Dual Potentiometers (Individual Project)

IoT Development Temperature Control

Context & Background

For this project, I built an IoT thermostat that can heat and cool more accurately. It was made for the IoT Recovery Exam. I needed to create a system that could control the temperature very precisely and cool things down faster when needed. I used two knobs: one to set the target temperature and another to control the speed of the cooling fan. This makes it easier to manage the temperature in different situations.

What I Built

  • Dual Control System: I made a system with two knobs. One for setting the temperature, and one for adjusting the fan speed.
  • Fast Cooling: I added a fan that you can speed up by turning a knob, so the system can cool down faster when needed.
  • Sending Data with MQTT: I set up a way for the OrangePi and Raspberry Pi Pico to send temperature data to each other using MQTT.
  • Live Temperature Reading: I made the system check the temperature all the time using a BMP280 sensor and adjust the heater or fan automatically.
  • Easy-to-Read Display: I connected an LCD screen to the Raspberry Pi Pico that shows the current and target temperatures.

Technologies Used

OrangePi Raspberry Pi Pico MQTT BMP280 Sensor PWM Control

Components & Implementation

The system was built using the following components, each serving a specific function in the thermostat system:

  • OrangePi: Served as the main controller, processing temperature data and managing system responses
  • BMP280 Sensor: Provided accurate temperature measurements communicated to the OrangePi
  • Dual Potentiometers: First potentiometer for setting target temperature via ADC, second for controlling cooling fan speed
  • Transistors: Two transistors implemented - one controlling current through the heating element, another managing fan operation
  • Raspberry Pi Pico: Received temperature data via MQTT and displayed information on the connected LCD screen
  • ThingSpeak Integration: Enabled cloud-based monitoring and data visualization through MQTT communications

What I Learned

Building the System

I learned how to put together different parts like sensors, fans, and controllers to make the thermostat work. I also set up a way to send temperature data using MQTT.
