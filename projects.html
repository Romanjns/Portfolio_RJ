Thomas Minder - Secure Hosting Platform

Cloud Infrastructure Security Kubernetes

Context & Background

The Thomas Minder project was developed as a secure hosting solution for a PHP application, created for our client Jochen. I, Siebe Moeskops, Gilles, Jorden Dillen, Sebbe Van Leemput, and Ward Smeyers, worked together on this project as part of SKILL 2.2. Our goal was to build a platform that combines strong security practices with an easy user experience, allowing clients to deploy PHP applications without worrying about security or infrastructure management. Key Features

  • User Authentication: Developed a secure system for user registration and login with role-based access.

  • Security Monitoring: Set up continuous security checks and automatic vulnerability scans.

IoT Thermostat with Enhanced Cooling System and Dual Potentiometers

IoT Development Temperature Control

Context & Background

This project involved designing and implementing an advanced IoT thermostat system with enhanced cooling capabilities. The system was created as part of the IoT Recovery Examination, addressing the need for precise temperature control with both heating and accelerated cooling functionalities. The thermostat utilizes dual potentiometers for independent control of target temperature and cooling fan speed, allowing for more efficient temperature management in varied environmental conditions.

Concrete Realizations

  • Dual Control System: Successfully implemented a system with two independent potentiometers - one for setting target temperature and another for controlling cooling fan speed.
  • Accelerated Cooling Mechanism: Designed and integrated a fan-based cooling system that can be manually adjusted to speed up temperature reduction when needed.
  • MQTT Communication: Established reliable data transmission between the OrangePi controller and Raspberry Pi Pico display unit using MQTT protocol.
  • Real-time Temperature Monitoring: Implemented continuous temperature sensing with BMP280 sensor and automatic adjustment of heating/cooling elements.
  • LCD Display Integration: Created a user-friendly interface displaying both target and current temperatures on an LCD screen connected to a Raspberry Pi Pico.

Technologies Used

OrangePi Raspberry Pi Pico MQTT BMP280 Sensor PWM Control

Components & Implementation

The system was built using the following components, each serving a specific function in the thermostat system:

  • OrangePi: Served as the main controller, processing temperature data and managing system responses
  • BMP280 Sensor: Provided accurate temperature measurements communicated to the OrangePi
  • Dual Potentiometers: First potentiometer for setting target temperature via ADC, second for controlling cooling fan speed
  • Transistors: Two transistors implemented - one controlling current through the heating element, another managing fan operation
  • Raspberry Pi Pico: Received temperature data via MQTT and displayed information on the connected LCD screen
  • ThingSpeak Integration: Enabled cloud-based monitoring and data visualization through MQTT communications

What I Learned

Temperature Measurement & Control

I learned how to use the BMP280 sensor to check the temperature all the time. I also learned how to make the system turn the heater or fan on or off to reach the temperature I wanted.

Cooling System

I learned how to control the fan speed using a second potentiometer. This helped cool things down faster when the temperature was too high.

Sending Data

I learned how to send temperature data using MQTT to a cloud website (ThingSpeak) and also show it on the Raspberry Pi Pico. This gave me two ways to see the information.

User Display

I learned how to set up an LCD screen with the Raspberry Pi Pico. It shows both the current and target temperatures so it's easy to see what’s happening right away.
